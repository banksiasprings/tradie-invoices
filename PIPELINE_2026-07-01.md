# Invoice App v90 → v93 Pipeline

**Approved by Steven 2026-07-01 after the closed-app ULTIMATE test PASSED end-to-end.** Field verified: app captured 09:37 start / 16:50 finish / 7.22h / $433 with the app never opened all day.

Notification firing is NOT a priority — Steven doesn't care if the tray notification fires. What matters: reliable data capture across many days without opening the app.

## Ship sequence

- **v90 — Multi-session queue + native event drainage** (flagship, same architectural shift, one bundle)
- **v91 — Round to nearest 15 min toggle** (Settings)
- **v92 — Self-diagnostic settings health check** (blocks Start Shift on critical fail)
- **v93 — Precision polish: concentric geofences + time interpolation** (only if field usage says needed)

Each ships to `main`, verified live before next spawns.

## v90 scope (flagship)

### Data model
- Replace single `activeDay` global with `unconfirmedSessions[]` array
- Each `WorkSession`: `{ id, site_id, start_time, end_time, status, gps_trace, lunch, rate, extra_labourer, notes, edited_by_user, rawStart, rawFinish }`
- `status`: `UNCONFIRMED` (default) | `CONFIRMED` | `REJECTED` | `MERGED`
- Only `CONFIRMED` sessions flow to invoice/billing/stats
- Unconfirmed preserved indefinitely — never auto-discarded

### Native drainage
- Move geofence handling from JS (Capacitor WebView-dependent) into a native Android **Service** or **WorkManager** expedited job
- GeofenceBroadcastReceiver (already fires when dead) → start short-lived foreground Service (`FOREGROUND_SERVICE_LOCATION` already declared)
- Native reads sites from SharedPreferences (already mirrored there)
- Native writes to Room DB (or extended SharedPrefs queue with `timer_state` record)
- Applies v89 accuracy/fresh-fix checks natively
- **JS becomes a read/repair layer**: on app open, reconciles localStorage with native-written state
- Dedup guard: extend v81 persisted flags into native layer so native + JS can't both start/stop

### Geofence handler
- ENTER → INSERT UNCONFIRMED session with `start_time` (v89 accuracy check applied natively)
- EXIT (v89 stop-confirmation) → UPDATE that session's `end_time`
- Multiple unconfirmed sessions co-exist — ENTER never blocks on prior unconfirmed
- Same-day rejoin < 90 min (`MERGE_WINDOW_MINUTES`, configurable) → MERGE into one session with lunch break

### UI
- **Today tab** — shows latest unconfirmed session (or empty state if all confirmed)
- **Log tab** — new "Review Backlog (N)" header with count of unconfirmed sessions
- Each row: three big buttons **Confirm / Adjust / Reject**
- Adjust opens the existing edit flow, stamps `edited_by_user: true`
- **Invoice + Stats** — count `CONFIRMED` only

### Edge cases
- Multi-site day → each site independent session
- Reboot mid-session → rely on Room DB persistence
- After-the-fact edits allowed, always stamp audit flag
- **Migration**: existing finalized day records → `CONFIRMED`. No data loss.

### Constraints
- Notification firing is NOT required — but leave the notification code path in place so Steven can enable in Android settings if he ever changes his mind
- Keep the existing JS replay path as fallback during rollout
- Test with a synthetic 5-day sequence before deploy
- Bump `APP_VERSION` to v90

## v91 scope (round to nearest 15)

- Settings toggle "Round times to nearest 15 min" — default OFF
- Applied at Confirm step in Review Backlog
- Rounds `start_time` and `end_time` to nearest :00/:15/:30/:45
- Keeps `rawStart` / `rawFinish` alongside for audit trail
- Field-proven need: Steven hand-rounds every day

## v92 scope (self-diagnostic)

- Run on app startup AND before every Start Shift
- Checks:
  - POST_NOTIFICATIONS granted (informational only if Steven has notifications OFF)
  - ACCESS_FINE_LOCATION granted
  - ACCESS_BACKGROUND_LOCATION = "Allow all the time"
  - `PowerManager.isIgnoringBatteryOptimizations()` = true
  - Doze / App-Standby bucket exempt
  - Manufacturer killers (branch on `Build.MANUFACTURER`) — Motorola Adaptive Battery, Samsung, Xiaomi, Oppo — deep-link their kill-list pages
  - Google Play services version recent enough for Geofencing API
  - RECEIVE_BOOT_COMPLETED registered
  - FOREGROUND_SERVICE_LOCATION declared (Android 14+)
- Each FAIL → big "❌ NEEDS FIXING" card + tap-to-open-settings using right Intent
- Block Start Shift while critical checks fail
- Show all as PASS/FAIL list in Settings → Health tab

## v93 scope (deferred precision polish)

- Concentric geofences (outer wake ring + inner event ring)
- Time interpolation between GPS polls for sub-30s crossing time
- Only build if field usage shows current precision isn't enough
