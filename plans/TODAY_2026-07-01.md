## v100 — Trip Log MVP scope

Ship as **v100** to signal "phase 2 major".

### Data model additions

New Firestore collection: `users/{uid}/trips/{tripId}`
```
{
  id: string,
  vehicle_id: string,
  category: 'business' | 'personal' | 'commute' | 'mixed' | 'unknown',
  start_time: timestamp,
  end_time: timestamp,
  start_lat: float,
  start_lng: float,
  end_lat: float,
  end_lng: float,
  polyline: [{lat, lng, t}, ...],       // GPS points every 30-60s
  distance_km: float,                    // Haversine sum
  duration_min: float,
  notes: string,
  linked_site_id: string?,               // if endpoint matches a saved site
  linked_invoice_id: string?,            // if attached to an invoice
  edited_by_user: boolean,
  created_at: timestamp
}
```

New Firestore doc: `users/{uid}/settings/vehicles/{vehicleId}`
```
{
  id: string,
  name: string,           // "My Ford Ranger"
  registration: string,
  make: string,
  model: string,
  year: number,
  cents_per_km: float,    // ATO rate default 0.78 for FY24-25
  is_default: boolean
}
```

### Auto-detection algorithm

- Monitor GPS in background (reuse v90 native location service)
- **Trip START**: sustained movement > 10 km/h for > 2 min AND not inside a geofenced work site
- **Trip END**: stopped (< 5 km/h) for > 5 min OR entered a geofenced work site
- **Polyline**: sample GPS every 30-60s during trip
- **Distance**: Haversine between consecutive points, sum
- **Category default**: `unknown` — user tags manually or auto-heuristics fill in later

### UI additions

- New **"Trips"** tab (5th tab: Today · Log · Invoice · Stats · **Trips** · Settings, or replace Settings with a gear icon on the top nav to save space)
- **Trips home**: today's trip list + total km + category pie
- **Individual trip card**: map polyline preview, times, km, tap to expand
- **Swipe left = personal, swipe right = business** on each trip card (with animation)
- **Weekly summary**: km per category, days driven
- **Vehicle setup** in Settings: add/edit/delete vehicles
